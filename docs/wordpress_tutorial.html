Validador de Anotações – Guia Passo a Passo

Da conversão de PDF/DOCX até a validação final no VAEST, com foco em linguistas e analistas que precisam produzir datasets confiáveis.

1. Visão Geral do Fluxo Contexto

O pipeline NET_TMA processa pares de textos para gerar datasets de simplificação textual:

[PDF/DOCX] → [Markdown] → [Anotação Manual] → [Parser]
→ [dataset_raw.json] → [VAEST] → [dataset_reviewed.json]
  • Conversão: transformar arquivos PDF/DOCX em Markdown estruturado.
  • Anotação: marcar estratégias no texto alvo usando tags (ex.: [SL+ termo]).
  • Parser: extrair amostras e criar dataset_raw.json.
  • Validação: revisar e corrigir no VAEST (GUI).
  • Exportação: gerar o dataset final validado.
2. Pré-requisitos Setup

Usuários Windows (vaest.exe)

  • Windows 10 ou superior
  • Pasta dist/ com vaest.exe e dataset_raw.json
  • Editor Markdown (VS Code, Notepad++, Typora)

3. Etapa 1 – Preparar Arquivos Entrada

Prepare três arquivos:

  1. Texto fonte (_st.md): original segmentado com IDs (01), (02), etc.
  2. Texto alvo (_tt.md): versão simplificada com anotações inline.
  3. Definições de tags (tab_est.md): tabela com códigos e descrições.

Use Markdown semântico (títulos #, ##) e mantenha a numeração sequencial para facilitar o alinhamento.

4. Etapa 2 – Converter PDF/DOCX Conversão

Opção A: Conversão manual (recomendada)

  • Pandoc: pandoc input.docx -o output.md
  • Word/LibreOffice: salvar como texto simples e aplicar Markdown
  • Ferramentas online: Pandoc Online, CloudConvert

Checklist após converter

  • Inserir IDs de parágrafo (01), (02)
  • Ajustar headings e revisar formatação
  • Validar que o texto mantém a estrutura original

Opção B: Script convert_inputs.py (placeholder)

python scripts/convert_inputs.py --docx arquivo.docx --output projeto_st.md

Atualmente apenas copia o arquivo; contribuições para a conversão real são bem-vindas.

5. Etapa 3 – Anotar Texto Alvo Tags

Tabela resumida de tags principais

CódigoNomeUso
RF+ReformulaçãoReescrever mantendo o sentido
SL+Simplificação LexicalSubstituir termo complexo
OM+OmissãoRemover informação
RP+Reconstrução de PeríodoReestruturar frases
RD+Reorganização DiscursivaMudar ordem das ideias
IN+InserçãoAdicionar conteúdo novo

Formato padrão: [TAG+ trecho original ou explicação]. Posicione a tag exatamente onde ocorreu a mudança.

  • SL+: "diferentes [SL+ díspares]".
  • RF+: "voltaram-se para o [RF+ inúmeras vozes...] patriotismo".
  • OM+: "texto [OM+ com detalhes removidos]".

Dicas: seja consistente, evite excesso de tags e valide em blocos menores.

6. Etapa 4 – Executar o Parser CLI 
cd caminho/para/net_tma/codebase
python -m parser.cli \
  --source meu_projeto_st.md \
  --target meu_projeto_tt.md \
  --tags tab_est.md \
  --output dataset_raw.json

Verifique se o número de amostras corresponde às tags e se os campos de parágrafo estão preenchidos.

Problemas? Consulte a seção "Solução de Problemas".

7. Etapa 5 – Validar no VAEST Revisão

Windows

  1. Copie dataset_raw.json para dist/.
  2. Abra vaest.exe.

macOS

  1. Copie dataset_raw.json para dist/.
  2. Right-click em vaest.app → Abrir → autorize em Segurança e Privacidade.
  3. Próximas execuções: duplo clique.

Destaques da interface

  • Painel esquerdo: lista de amostras com status.
  • Painel direito: detalhes, checkbox "Necessita revisão humana", notas, revisor, histórico.
  • Toolbar: filtros por tag/status, busca global, botões Recarregar/Salvar.

Fluxo sugerido

  1. Filtrar "Necessita revisar".
  2. Comparar contexto, fonte e alvo.
  3. Ajustar status, inserir notas e iniciais.
  4. Salvar incrementalmente (dataset_reviewed_parcial.json).
8. Etapa 6 – Exportar Dataset Final Entrega
  1. Use Arquivo → Salvar como... e nomeie como dataset_reviewed.json.
  2. Verifique campos adicionais: necessita_revisao_humana, motivo_revisao, reviewer, history.
  3. Aplicações: treino de modelos, análise linguística, métricas de qualidade, recursos lexicais.

9. Solução de Problemas Suporte

Arquivos não encontrados

  • Use caminhos absolutos e confirme extensões .md.

JSON vazio

  • Cheque formato [TAG+ texto] e consistência com tab_est.md.

Alinhamento incorreto

  • Garanta IDs sequenciais e headings equivalentes.

VAEST não abre

  • Windows: execute vaest.exe --headless --dataset dataset_raw.json.
  • macOS: ./vaest.app/Contents/MacOS/vaest --headless --dataset dataset_raw.json e autorize o app.

Erro ao salvar

  • Salve em pastas com permissão, feche outros programas, use "Executar como administrador".
